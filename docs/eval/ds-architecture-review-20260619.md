## ❌ DS 指出的 8 项问题

### 架构缺陷

| # | 问题 | 严重度 |
|---|------|--------|
| 1 | **单 prompt 问 5 种风险** — QASecClaw 证明多 Agent 精确度高 23% | 🔴 高 |
| 2 | **无反馈闭环** — 用户纠正不回灌到规则/ML/提示词 | 🔴 高 |
| 3 | **固定 300 行分块无重叠** — 跨函数边界逻辑漏洞漏检 | 🟡 中 |
| 4 | **ML filter 太窄** — 只有字面量检测，缺 API 使用模式分类 | 🟡 中 |
| 5 | **无评测框架** — 没有 precision/recall/F1，改进靠猜测 | 🔴 高 |
| 6 | **DS Review 和 Heuristic 用同一模型** — 分类器知道启发式输出，有偏见 | 🟡 中 |
| 7 | **7 条规则只能手动修复** — 开发体验割裂 | 🟢 低 |
| 8 | **无主动学习** — 不接受用户反馈来调优 | 🟡 中 |

### 文献中漏掉的洞察

**QASecClaw 启示：**
- 每个 Agent 应接收 **结构化元数据**（AST 节点 ID、数据流路径）
- 需要 **协调 Agent** 去重和优先级排序

**SAST-Genius 启示：**
- LLM 应只在 SAST 已标记的范围内确认/拒绝，主动生成新发现 FP 率会暴涨 ⚠️
- 用 **2 个本地模型共识**（保守+激进）提升精确度 18%

**Local LLM 启示：**
- 用 **CVE 安全补丁数据集** 微调本地模型，TOCTOU 召回率翻倍
- **滑动窗口 20% 重叠** 优于固定分块
- **置信度校准**（temperature scaling）大幅降低过度自信的误报

## 🎯 DS 推荐 Top 3 改进

### 1. 单 prompt → 多 Agent 管线
- 5 个专用 prompt/Agent，各管一种风险
- 协调 Agent 合并去重
- **并行执行**避免延迟
- 预期：F2 +23%

### 2. 反馈闭环
- 用户接受/拒绝标记 → 回灌 SAST 规则参数
- 项目级置信度自适应
- LLM 发现 → 更新提示词

### 3. 评测框架
- 用 Juliet/CWE 测试集
- 每次改进跑 precision/recall/F1
- 在 README 公开评测数据 → 建立社区信任

---

## 🚨 最关键的警告

> **SAST-Genius 的核心洞察**：LLM 应只确认 SAST 已标记的发现，而非凭空生成新发现——否则 FP 率会暴涨。

这对我们的 `--labs` 模块是直接挑战。我们正在做的"发现规则发现不了的漏洞"与文献结论冲突。
建议：在 [Labs] 区更显著地标注 "EXPERIMENTAL — HIGH FP RISK"，并考虑限定 LLM 只审查 SAST 已标记区域而非全文件。

---

> 评审结论：管线方向正确，但需要 (1) 多 Agent 分治、(2) 评测框架、(3) 反馈闭环。LLM 凭空发现漏洞需谨慎对待。
